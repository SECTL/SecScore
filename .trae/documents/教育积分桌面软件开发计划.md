# 教育积分桌面软件开发计划

## 项目概述

使用Tauri 2.0 + React + Rust单体架构开发教育积分管理桌面应用，集成SQLite本地数据库，无网络依赖。

## 开发阶段与任务

### Phase 1: 核心基础

1. **项目初始化**

   * 初始化Tauri 2.0项目

   * 配置React前端环境

   * 设置Rust后端依赖

2. **数据库配置**

   * 集成SQLite数据库

   * 创建数据库表结构

   * 配置sqlx连接

3. **学生管理基础功能**

   * 学生数据结构定义

   * 增删改查API实现

   * 学生列表UI组件

4. **积分操作基础功能**

   * 积分操作API

   * 积分操作UI界面

   * 基本表单验证

5. **权限验证基础**

   * 用户表结构设计

   * 密码哈希实现

   * 登录界面开发

6. **基础UI框架**

   * 主窗口布局

   * 导航栏设计

   * 侧边栏菜单

### Phase 2: 核心功能

1. **积分记录系统**

   * 积分记录API

   * 记录查询功能

   * 历史记录UI

2. **排行榜功能**

   * 排行榜API

   * 排序与筛选

   * 排行榜UI

3. **权限系统完善**

   * 三级权限控制

   * 权限中间件

   * 功能访问控制

4. **主题系统**

   * 预设主题配置

   * 主题切换功能

   * 主题持久化

5. **数据备份恢复**

   * 备份API实现

   * 恢复功能

   * 自动备份机制

6. **用户界面优化**

   * 响应式设计

   * 交互体验优化

   * 错误处理

### Phase 3: 增强功能

1. **批量导入导出**

   * CSV导入功能

   * 数据导出功能

2. **高级搜索筛选**

   * 学生搜索优化

   * 记录筛选功能

3. **快捷键支持**

   * 常用操作快捷键

   * 键盘导航

   * 托盘菜单

### Phase 4: 优化完善

1. **性能优化**

   * 数据库查询优化

2. **代码质量检查**

   * Rust Clippy检查

   * TypeScript类型检查

   * 代码格式化

3. **打包发布**

   * 多平台打包配置

## 技术栈与依赖

### 前端

* React 18

* TypeScript

* Vite

* Tailwind CSS (可选)

### 后端

* Rust

* Tauri 2.0

* SQLite

* sqlx

* argon2id (密码哈希)

* tokio (异步运行时)

* tracing (日志)

## 项目结构

```
edu-points/
├── src-tauri/          # Rust后端
│   ├── src/
│   │   ├── main.rs     # 应用入口
│   │   ├── commands/   # 功能模块
│   │   ├── database/   # 数据库操作
│   │   ├── crypto/     # 加密功能
│   │   └── utils/      # 工具函数
│   └── tauri.conf.json
└── src/                # React前端
    ├── components/     # UI组件
    ├── pages/         # 页面组件
    └── hooks/         # 自定义Hooks
```

## 开发流程

1. 每周完成一个开发阶段
2. 每个功能模块完成后进行测试
3. 定期提交代码并记录开发日志
4. 遵循代码规范和最佳实践
